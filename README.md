# kata-machine

Daily DSA practice with spaced repetition.

### Setup

```bash
npm install
```

### Workflow

```bash
npm run pick              # pick 5 katas (SR-prioritized)
npm run generate          # create stub files
# ... solve the katas in src/session{...}/
npm run complete-day      # run tests + self-assess (0-5)
```

### Commands

| Command | Description |
|---|---|
| `npm run pick` | Pick 5 SR-prioritized katas (no consecutive-day repeat) |
| `npm run pick -- --count 3` | Pick N katas instead of 5 |
| `npm run pick -- --katas BubbleSort,QuickSort` | Pick specific katas |
| `npm run generate` | Create stub files in `src/session{...}/` |
| `npm run complete-day` | Run jest on active katas, then prompt 0-5 per kata |
| `npm run review` | Just self-assess (no tests) |
| `npm run stats` | Show progress: reviewed, due, streak, buckets |

### Spaced Repetition (SM-2)

After each practice session, rate yourself 0-5 per kata:

- **0** — complete blackout
- **1** — incorrect, recalled with difficulty
- **2** — incorrect, but close
- **3** — difficult but recalled correctly
- **4** — correct after some hesitation
- **5** — perfect recall

Scores ≥ 3 grow the review interval; scores < 3 reset it. A minimum 3-day interval prevents over-repetition. Data stored in `.review-data.json`.

### pick logic

1. Katas with `next_review` ≤ today are sorted by easiness (lowest first)
2. If fewer than N are due, fills with unseen or least-recently-reviewed katas
3. Previous day's picks are excluded

### Config

Active katas are stored in `.active.json` (auto-generated by `npm run pick`).

### All Katas

50+ katas covering arrays, linked lists, trees, graphs, sorting, dynamic programming, and more. Defined in `scripts/dsa.js`.
